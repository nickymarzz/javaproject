# Security Policy

## Supported Versions

This project does not currently maintain multiple supported release branches. Security fixes, when available, are expected to be applied to the latest code in the default branch.

## Reporting a Vulnerability

Please do not report security vulnerabilities through public issues.

Instead, contact the project maintainers privately and include:

- A clear description of the issue
- Steps to reproduce or validate it
- The affected component or file, if known
- Any suggested mitigation or fix

If you are adopting this repository publicly, replace this section with your preferred confidential contact method, such as a dedicated security email address.

Example placeholder:

- Security contact: `security@example.com`

Maintainers should acknowledge receipt as soon as possible, investigate the report, and coordinate disclosure after a fix is available.
